# Stream notifier bot

Telegram-бот отправляет в выбранные группы уведомления о начале Twitch и
YouTube- и Kick-эфиров. Каналы добавляются в личке с ботом, а группа для уведомлений
выбирается кнопкой.

## Переменные окружения

Скопируйте `.env.example` в секреты хостинга и заполните:

- `BOT_TOKEN` — токен от [BotFather](https://t.me/BotFather).
- `TWITCH_CLIENT_ID` и `TWITCH_CLIENT_SECRET` — приложение в [Twitch Developer Console](https://dev.twitch.tv/console/apps).
- `YOUTUBE_API_KEY` — ключ Google Cloud проекта с включённым YouTube Data API v3.
- `KICK_CLIENT_ID` и `KICK_CLIENT_SECRET` — данные приложения из
  [Kick Developer Portal](https://kick.com/settings/developer).

База SQLite хранится в `DB_PATH`. Для Docker смонтируйте `/app/data` как постоянный том, иначе состояние подписок будет потеряно при пересоздании контейнера.

## Настройка в Telegram

1. Добавьте бота администратором в нужный канал или группу. Для этого не нужно
   писать никаких команд в самом канале: бот автоматически запомнит чат в момент
   добавления.
   Если это Telegram-форум, откройте нужную тему и отправьте в ней `/topic`.
   Бот запомнит тему и будет отправлять уведомления именно туда. Список тем
   получить через Telegram Bot API нельзя, поэтому выбор выполняется внутри
   самой нужной темы.
2. Откройте личный чат с ботом и используйте появившееся меню:
   - **Добавить канал** — выбрать Twitch, YouTube или Kick, прислать ссылку и
     выбрать получателя уведомлений.
   - **Мои подписки** — открыть сведения о канале или удалить его с
     подтверждением.
   - **Проверить эфиры** — выполнить проверку немедленно.
   - **Оформление** — изменить заголовок и описание уведомления, загрузить свою
     картинку или удалить её, добавить любые кнопки со ссылками (включая
     Discord), выбрать источник автоматического превью, включить блюр, посмотреть
     текущие настройки, выбрать эмодзи и цвет кнопок или отправить себе пример
     уведомления.
     При добавлении кастомной кнопки укажите номер строки: кнопки с одинаковым
     номером будут показаны в одной группе.
     Кастомные эмодзи можно отправить боту при настройке кнопки. Для их показа
     владелец бота должен иметь Telegram Premium либо бот должен иметь
     дополнительный username, купленный на Fragment.
     Для кнопок доступны встроенные стили Telegram: синий, зелёный и красный.
3. В описание можно добавить правила, ссылки и другую постоянную информацию;
   оно будет показано над списком начавшихся эфиров. При изменении заголовка
   и описания доступны подстановки `{count}` — число новых эфиров, и `{time}` —
   время начала эфира по Москве.
   Для каждой платформы также доступны название и категория: `{titleYT}`,
   `{titleTwich}`, `{titleKick}`, `{categoryYT}`, `{categoryTwich}`,
   `{categoryKick}`. Написание `{titleTwitch}` и `{categoryTwitch}` также
   поддерживается.
4. Чтобы отменить незавершённый шаг, нажмите **Отмена** в нижнем меню.

Команды `/add`, `/chats`, `/list`, `/remove`, `/check`, `/template` и
`/preview` сохранены как запасной вариант.

Для Kick добавьте канал через меню либо командой:
```
/add kick https://kick.com/streamer
```

Первый опрос после добавления запоминает текущий статус канала, но не уведомляет об уже идущем эфире. Уведомление придёт при следующем переходе канала из офлайна в эфир.

Twitch и Kick проверяются каждые 90 секунд, YouTube — каждые 300 секунд.
Уведомление отправляется сразу после обнаружения нового эфира. Если второй
эфир будет обнаружен позднее, бот отредактирует уже отправленное уведомление,
добавив информацию о нём. Уведомление содержит загруженную через
`/preview` картинку. Для автоматического превью можно выбрать Twitch, YouTube,
Kick или автовыбор. Twitch-карточка содержит превью эфира, аватар стримера и
обложку игры; при блюре аватар и обложка категории остаются чёткими. Для
YouTube и Kick блюр применяется к фону превью.

## Лимиты YouTube

Проверка YouTube использует `search.list` с фильтром `eventType=live`; этот
вызов расходует API-квоту Google. При большом количестве YouTube-каналов
увеличьте `YOUTUBE_POLL_INTERVAL_SECONDS` и проверьте квоту проекта в Google
Cloud Console.
